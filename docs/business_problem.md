# Problema do Negocio

## Contexto

Como entregadora de aplicativos, observei uma recorrência no fluxo de mercados: pedidos marcados como "prontos para retirada" frequentemente ainda estavam em etapas internas da operação.

Essa diferença entre o status exibido no aplicativo e o status real do pedido gera:

- Tempo de espera para entregadores
- Cancelamentos evitáveis
- Perda de produtividade
- Congestionamento nos pontos de coleta
- Experiência negativa para entregadores e operação

## Observação em Campo

No fluxo de mercados o pedido normalmente passa por diferentes etapas:

1. Coleta dos itens
2. Conferência / prevenção de perdas
3. Embalagem
4. Liberação para retirada

Porém, em alguns cenários, o status de "pedido pronto" pode ser exibido antes da conclusão da última etapa operacional.


## Questão do negócio
Quais fatores aumentam a probabilidade de um entregador chegar para retirar um pedido marcado como pronto e encontrá-lo ainda em preparação?
